MODELS.PY IN FASTAPI PROJECTS

1. Purpose of models.py
- Defines data models using Pydantic for request validation and response formatting.
- Ensures that incoming data (from clients) matches expected types and structures.
- Provides schemas for outgoing data (responses) to maintain consistency.
- Can also contain ORM models (e.g., SQLAlchemy) for database representation.
- Acts as the bridge between raw data (JSON, DB rows) and Python objects.

2. Typical Structure of models.py

from pydantic import BaseModel

# Request model
class UserCreate(BaseModel):
    username: str
    email: str
    password: str

# Response model
class UserResponse(BaseModel):
    id: int
    username: str
    email: str

Explanation:
- from pydantic import BaseModel → imports the base class for creating Pydantic models.
- class UserCreate(BaseModel) → defines the schema for creating a new user (input validation).
- username, email, password → required fields with type hints.
- class UserResponse(BaseModel) → defines the schema for returning user data (output formatting).
- id, username, email → fields included in the response.

3. Summary
- models.py centralizes data definitions for both requests and responses.
- Pydantic automatically validates and converts data types.
- Using models improves reliability, readability, and documentation of the API.
- Models can be reused across routes, services, and tests.

4. Best Practices/Standards with models.py
- Separate request and response models to avoid exposing sensitive fields (e.g., password).
- Use descriptive names (UserCreate, UserUpdate, UserResponse) for clarity.
- Leverage Pydantic features like default values, validators, and nested models.
- Keep models.py focused on schemas; place database ORM models in a dedicated database/ folder.
- Document models with field descriptions for better auto-generated docs.
- Use type hints consistently to ensure validation and conversion.
- Avoid mixing business logic inside models; they should only define structure and validation.
- Reuse models across routers and services to maintain consistency.
