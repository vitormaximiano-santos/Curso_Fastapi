ROUTE: @order_router.post("/pedido/cancelar/{id_pedido}")

Code:
@order_router.post("/pedido/cancelar/{id_pedido}")
async def cancelar_pedido(id_pedido: int, session: Session = Depends(pegar_sessao),usuario:Usuario = Depends(verificar_token)):
    pedido = session.query(Pedido).filter(Pedido.id == id_pedido).first()
    if not pedido:
        raise HTTPException(status_code=404, detail="Pedido não encontrado")
    
    if not Usuario.admin and Usuario.id != pedido.usuario:
        raise HTTPException(status_code=401, detail="Você não tem permissão para cancelar esse pedido")
    
    pedido.status = "cancelado"
    session.commit()
    return {
        "Mensagem": f"Pedido número {pedido.id} foi cancelado com sucesso",
        "pedido": pedido
        }

   - Defines a POST route at /pedidos/pedido/cancelar/{id_pedido}.
   - POST is chosen because the request modifies data (cancels an order).
   - {id_pedido}: path parameter representing the ID of the order to cancel.

2. "async def cancelar_pedido(id_pedido: int, session: Session = Depends(pegar_sessao), usuario: Usuario = Depends(verificar_token))"
   - async def: asynchronous function, required by FastAPI for route handlers.
   - id_pedido: integer parameter taken from the URL path, identifies the order to cancel.
   - session: SQLAlchemy Session object.
       - Obtained via Depends(pegar_sessao), which injects a database session into the route.
   - usuario: Usuario object.
       - Obtained via Depends(verificar_token).
       - verificar_token: validates the JWT token and retrieves the authenticated user.
       - Ensures only authenticated users can attempt to cancel orders.

3. "pedido = session.query(Pedido).filter(Pedido.id == id_pedido).first()"
   - Queries the Pedido table for an order with the given ID.
   - .filter(Pedido.id == id_pedido): filters by order ID.
   - .first(): returns the first matching order or None if not found.
   - pedido: variable holding the order object.

4. "if not pedido:"
   - Checks if the order was not found in the database.
   - If true → raises an exception.

5. "raise HTTPException(status_code=404, detail='Pedido não encontrado')"

   - detail: message explaining the error.
   - Prevents cancellation of non-existent orders.

---

6. "if not Usuario.admin and Usuario.id != pedido.usuario:"
   - Checks if the current user has permission to cancel the order.
   - Conditions:
       - If the user is not an admin (Usuario.admin is False).
       - AND the user’s ID does not match the order’s owner (pedido.usuario).
   - If both conditions are true → user is unauthorized.

7. "raise HTTPException(status_code=401, detail='Você não tem permissão para cancelar esse pedido')"
   - Raises an HTTPException with status code 401 (Unauthorized).
   - detail: message explaining the error.
   - Ensures only admins or the order’s owner can cancel the order.

---

8. "pedido.status = 'cancelado'"
   - Updates the order’s status to "cancelado" (canceled).
   - Marks the order as canceled in the database.

9. "session.commit()"
   - Commits the transaction.
   - Saves the updated order status permanently in the database.

---

10. "return {'Mensagem': f'Pedido número {pedido.id} foi cancelado com sucesso', 'pedido': pedido}"
    - Returns a JSON response confirming the cancellation.
    - Includes:
        - Mensagem: success message with the order ID.
        - pedido: the order object (with updated status).

SUMMARY
- This route cancels an order by ID.
- It requires authentication via verificar_token.
- Checks if the order exists.
- Ensures only admins or the order’s owner can cancel it.
- Updates the order status to "cancelado" and saves it.
- Returns a success message with the order details.
