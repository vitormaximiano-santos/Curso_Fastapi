ROUTE: @order_router.get("/pedido/{id_pedido}")

Code:
@order_router.get("/pedido/{id_pedido}")
async def visualizar_pedido(id_pedido: int, session: Session = Depends(pegar_sessao),usuario:Usuario = Depends(verificar_token)):
    pedido = session.query(Pedido).filter(Pedido.id == id_pedido).first()
    if not pedido:
        raise HTTPException(status_code=400, detail="Pedido não encontrado")
    
    if not Usuario.admin and Usuario.id != pedido.usuario:
        raise HTTPException(status_code=401, detail="Você não tem permissão para visualizar esse pedido")
    
    return {
        "Quantidade de itens": len(pedido.itens),
        "Pedido": pedido
    }
---

1. "@order_router.get('/pedido/{id_pedido}')"
   - Defines a GET route at /pedidos/pedido/{id_pedido}.
   - GET is chosen because the request retrieves data (details of a specific order).
   - {id_pedido}: path parameter representing the ID of the order to view.

2. "async def visualizar_pedido(...)"  
   - async def: asynchronous function, required by FastAPI for route handlers.
   - Parameters:
     - id_pedido: integer path parameter, identifies the order to view.
     - session: SQLAlchemy Session object.
         - Obtained via Depends(pegar_sessao), which injects a database session into the route.
     - usuario: Usuario object.
         - Obtained via Depends(verificar_token).
         - verificar_token: validates the JWT token and retrieves the authenticated user.

---

3. "pedido = session.query(Pedido).filter(Pedido.id == id_pedido).first()"
   - Queries the Pedido table for an order with the given ID.
   - .filter(Pedido.id == id_pedido): filters by order ID.
   - .first(): returns the first matching order or None if not found.
   - pedido: variable holding the order object.

---

4. "if not pedido:"
   - Checks if the order was not found in the database.
   - If true → raises an exception.

5. "raise HTTPException(status_code=400, detail='Pedido não encontrado')"
   - Raises an HTTPException with status code 400 (Bad Request).
   - detail: message explaining the error.
   - Prevents viewing non-existent orders.

---

6. "if not Usuario.admin and Usuario.id != pedido.usuario:"
   - Checks if the current user has permission to view the order.
   - Conditions:
       - If the user is not an admin (Usuario.admin is False).
       - AND the user’s ID does not match the order’s owner (pedido.usuario).
   - If both conditions are true → user is unauthorized.

7. "raise HTTPException(status_code=401, detail='Você não tem permissão para visualizar esse pedido')"
   - Raises an HTTPException with status code 401 (Unauthorized).
   - detail: message explaining the error.
   - Ensures only admins or the order’s owner can view the order.

---

8. "return {'Quantidade de itens': len(pedido.itens), 'Pedido': pedido}"
   - Returns a JSON response with details of the order.
   - Includes:
       - Quantidade de itens: number of items in the order (length of pedido.itens).
       - Pedido: the order object itself.

---

SUMMARY
- This route retrieves details of a specific order by ID.
- It requires authentication via verificar_token.
- Checks if the order exists.
- Ensures only admins or the order’s owner can view it.
- Returns the order details, including the number of items and the order object.
